# zstdfs-mac-kext

A macOS `decmpfs` compression kext adding **zstd** as transparent-decompression
types 200/201, plus a userspace compressor. Intended for pre-compressing macOS
VM images at distribution time (zstd level 19/22) with transparent kernel
decompression at runtime.

> ## Status: PARKED — not viable on Apple Virtualization Framework VMs
>
> The kext is **code-complete and verified correct** against the real kernel
> ABI, but it cannot be loaded in the intended target (an Apple Virtualization
> Framework / UTM macOS guest). **Third-party kexts are unsupported in AVF VMs
> by Apple's design**: iBoot refuses to admit an Auxiliary Kernel Collection
> (AuxKC) into a VM's memory map, and Apple closed the radar with *"3rd party
> kext (auxKC) isn't supported on macOS VMs."*
>
> The only known workaround requires binary-patching the boot chain —
> **including the host's `Virtualization.framework` (`AVPBooter`), with host SIP
> disabled** — plus the guest `iBoot`/`kernelcache`. Modifying the host's system
> framework to load a kext into a VM is not something that can be asked of a
> consumer machine, so the project is parked here. See [References](#references).
>
> If Apple ever supports third-party kexts in AVF guests, this is worth
> resuming — the hard part (the kext itself) is done.

## What this is

macOS uses `decmpfs` for transparent filesystem compression — files with
`UF_COMPRESSED` set are decompressed on the fly by the kernel. Apple ships types
3–14 (DEFLATE, LZVN, LZFSE, LZBITMAP). This project adds types 200/201 using
zstd, which compresses meaningfully better than the system algorithms at ultra
levels (~6% of raw size beyond LZFSE on representative arm64 binaries).

**Target use case**: bundling Rosetta, dylibs, and frameworks into a macOS VM
image for distribution, pre-compressed offline (slow is OK), decompressed
transparently at read time (fast). **Requires a no-SIP VM and an unsigned kext.**

## What actually got built and verified

- **Userspace compressor** (`tool/`, Swift) — writes type 200/201 decmpfs
  attributes.
- **The kext** (`kext/`) — IOService that registers zstd decompressors for types
  200/201. Builds clean for arm64e, ad-hoc signs, and passes `kmutil` validation.
- **The hard part: the private `decmpfs` ABI, recovered empirically** from
  `KDK_26.4_25E246`'s `kernel.release.vmapple` and its DWARF, since
  `bsd/sys/decmpfs.h` is `KERNEL_PRIVATE` and not shipped. See
  [`kext/src/decmpfs_private.h`](kext/src/decmpfs_private.h) — it documents the
  exact struct layout, the single vector-based `fetch` callback signature, the
  per-field arm64e PAC discriminators (key IA, no address diversity), the
  version check `(v|2)==3`, and the valid type range (0–254).
- **Runtime symbol resolution** ([`kext/src/kernel_sym.c`](kext/src/kernel_sym.c))
  — `register_decmpfs_decompressor` lives only in `com.apple.kpi.private`, which
  forces a `com.apple.*` bundle id that the AuxKC loader then rejects. To dodge
  that, the symbols are resolved by fixed offset from a public KPI anchor
  (`IOLog`), guarded by a function-prologue byte check so a wrong offset bails
  cleanly instead of panicking, and called via a raw `blr`.

What was **not** reached: a live load / end-to-end decompression test, blocked
solely by the AVF limitation above — not by anything in the code.

## Type numbering

| Type | Storage | Notes |
|------|---------|-------|
| 200  | `com.apple.decmpfs` xattr (inline) | small files |
| 201  | `com.apple.ResourceFork` xattr, 256 KB chunks | larger files (decompressor stubbed — needs rsrc-fork read) |

Each chunk (type 201) is a complete zstd frame stored verbatim. Type 200 stores
one frame inline after the 16-byte decmpfs header.

## Build

```bash
cd kext
make vendor-zstd     # populate kext/zstd/ from the Linux lib/zstd kernel port
make                 # build DecmpfsZstd.kext (arm64e)
```
The build uses a KDK if one ships full headers, otherwise the CLI Tools SDK.

## Project layout

```
kext/
  DecmpfsZstd.kext/Contents/Info.plist   kext bundle metadata
  src/
    service.cpp        IOService subclass; registers types 200/201
    decmpfs_private.h  recovered private decmpfs ABI (the reference artifact)
    kernel_sym.c/.h    runtime resolution of the private registration symbols
    type200.c          inline (xattr) zstd fetch callback
    type201.c          rsrc-fork fetch (stubbed)
    zstd_ctx.c/h       pool of pre-allocated ZSTD_DCtx workspaces
  zstd/                vendored zstd (regenerated by scripts/vendor-zstd.sh)
tool/                  Swift userspace compressor (compress/info/verify/tree)
scripts/
  vendor-zstd.sh       fetch + adapt the zstd kernel port
  compress-vm.sh       orchestrate a VM-image compression pass
  check-vm.sh          inspect a VM's kext-loading prerequisites (read-only)
```
